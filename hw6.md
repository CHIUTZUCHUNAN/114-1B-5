# 流程圖
```mermaid
flowchart LR
  A[登入/身分確認] --> B[考前自檢]
  B --> C[監控儀表板（監考員）]
  B --> S[考生監考畫面（考生端）]
  C --> D[事件詳情/證據檢視]
  C --> E[門檻/參數設定]
  C --> F[匯出與報表]
  F --> G[會話摘要報表]
  F --> H[事件明細報表]
```
# 分鏡表
## 【1】登入/身分確認
* 目的：驗證身分與角色，建立會話前置。
* 主要使用者：監考員、考生

![登入/身分確認](登入身分確認.png)

## 【2】考前自檢
* 目的：檢測裝置可用性與環境品質
* 主要使用者：監考員、考生

![考前自檢](考前自檢.png)

## 【3】監控儀表板（監考員）
* 目的：即時監控、事件列表與狀態掌握
* 主要使用者：監考員

![監控儀錶板](監控儀錶板.png)

## 【4】事件詳情/證據檢視
* 目的：檢視單一事件之證據與參數
* 來源：儀表板點擊事件
  
![事件詳情_證據檢視](事件詳情_證據檢視.png)
